## REFERENCIA VISUAL DO PROJETO

Este projeto segue um padrao visual premium inspirado em:

- interfaces modernas de hotel / hospitalidade
- design minimalista e sofisticado
- uso de glassmorphism (blur + transparencia)
- tipografia grande e impactante
- muito respiro (spacing bem trabalhado)
- poucos elementos, porem bem organizados
- foco visual claro (sem poluicao)

Evitar:
- aparencia de template generico
- excesso de bordas
- excesso de caixas
- layout pesado
- visual "quadrado" ou travado

## FRONTEND (UX/UI)

### Principios
- Hierarquia visual clara
- Espacamento consistente
- Alinhamento preciso
- Tipografia bem definida
- Design leve e elegante
- Aparencia premium

### Sempre revisar:
- padding e margin
- alinhamento de elementos
- proporcao entre blocos
- consistencia entre componentes
- contraste de texto
- responsividade (mobile-first)
- estados de interacao (hover, focus, active)

### Visual
- Preferir menos elementos com mais qualidade
- Evitar poluicao visual
- Usar transparencia e blur com elegancia
- Evitar blocos pesados e excessivamente delimitados

## UX (EXPERIENCIA DO USUARIO)

- O usuario deve entender a interface em poucos segundos
- Priorizar clareza sobre quantidade de informacao
- Evitar esforco cognitivo desnecessario
- Destacar acoes principais
- Garantir navegacao intuitiva

---

## REGRAS PARA INTERFACES ADMIN

### Qualidade esperada
- Interface limpa, organizada e profissional
- Aparencia de produto real, nao de sistema improvisado

### Estrutura
- Agrupar informacoes de forma logica
- Evitar excesso de containers dentro de containers
- Reduzir bordas desnecessarias
- Usar espacamento para separar, nao apenas linhas

### Acoes (CRITICO)
- Botoes e acoes devem ser coerentes e alinhados
- Nunca deixar botao principal e acao destrutiva "soltos"
- Sempre agrupar acoes no mesmo contexto visual

Errado:
[Salvar]        [Excluir] (solto)

Correto:
[Salvar] [Excluir]
ou
[Salvar]    (Excluir alinhado como acao secundaria clara)

### Hierarquia
- Acao principal (Salvar) deve ter destaque
- Acao destrutiva (Excluir) deve ser visivel, mas secundaria

### Inputs
- Campos importantes devem ter mais destaque
- Campos auxiliares devem ter menos peso visual

### Revisao obrigatoria
Antes de finalizar qualquer tela admin, verificar:

- Esta limpo ou poluido?
- Esta alinhado ou "quebrado"?
- Esta moderno ou parece sistema antigo?
- As acoes estao bem organizadas?
- Da vontade de usar ou parece improvisado?

Se estiver ruim -> refazer layout.
